# 产品定位与角色覆盖 v0.1

> **一句话定位**：一个人的内容团队
>
> **目标用户**：内容创业者 / 做内容的人（从炸油条的店铺推广到专业写手、电商卖家、MCN 从业者）
>
> **核心主张**：替代内容创业者需要雇的 10+ 个岗位——从选题到成稿、从数据分析到投放执行，一个对话窗口搞定。

---

## 一、产品本质

对话驱动的 AI 内容团队。软件本身是通道，用户通过对话向"团队"下达任务、传递素材、获取产出。

不是 IDE，不是编辑器，不是工作流平台。是**人**——一群懂内容、懂运营、懂数据的 AI 队友。

---

## 二、底层能力矩阵

产品能力不仅限于"LLM 对话"，而是建立在以下能力栈之上：

| 能力层 | 具体能力 | 当前状态 |
|--------|---------|---------|
| **LLM 对话** | 多模型切换、长上下文、流式输出 | ✅ 已有 |
| **知识库（KB）** | 风格库/素材库、抽卡、向量检索、风格仿写 | ✅ 已有 |
| **联网** | web.search / web.fetch | ✅ 已有 |
| **文件读写** | doc.read / doc.write / project 操作 | ✅ 已有 |
| **Skills** | 可插拔技能包——每个 Skill 是一组 prompt + 工具组合，按意图自动激活 | ✅ 框架已有，待扩展 |
| **MCP（Model Context Protocol）** | 标准化外部工具/服务接入协议，第三方能力即插即用 | 🔧 待接入 |
| **Bash / 沙箱** | 在隔离沙箱中执行任意脚本 | 🔧 待接入 |
| **无头浏览器** | Playwright/Puppeteer，网页操作、截图、数据抓取 | 🔧 待接入 |
| **定时任务** | cron 式调度（竞品监控、定时发布等） | 🔧 待接入 |
| **平台 API 对接** | 聚光、巨量引擎、小红书开放平台等广告/内容 API | 🔧 待接入 |
| **视觉能力** | 图片理解/生成（接入视觉模型如 nano/banana pro） | 🔧 待接入 |
| **办公套件 Skills** | Excel/Word/PPT 生成与编辑（产出成品文件而非 Markdown） | 🔧 待开发 |
| **记忆系统** | 对话/决策/素材向量化 → 无限上下文 + 团队共享记忆 | 🔧 待开发 |

### 能力上限原则

有 bash + 沙箱 + 无头浏览器 = 有无限可能。理论上能做任何一台电脑能做的事，包括但不限于：
- 爬取竞品账号数据并生成分析报告
- 调用聚光 API 自动创建/调整广告计划
- 定时监控关键词排名变化
- 自动截图竞品内容并提取文案
- 批量生成多平台适配的内容变体

### Skills 与 MCP：能力扩展机制

**Skills（内置技能包）**——每个 Skill = 一组专用 prompt 片段 + 工具权限 + 触发条件，按用户意图自动激活：

| Skill | 用途 | 产出物 |
|-------|------|--------|
| `style_imitate` | 风格仿写闭环 | 符合用户调性的文案 |
| `web_topic_radar` | 全网热点/素材雷达 | 选题盘点报告 |
| `corpus_ingest` | 语料导入与抽卡 | 风格库/素材库 |
| `excel`（待开发） | 数据整理、表格生成、数据分析 | .xlsx 文件 |
| `word`（待开发） | 长文档排版、方案书、合同模板 | .docx 文件 |
| `ppt`（待开发） | 汇报演示、方案 deck、内容总结 | .pptx 文件 |
| `image_gen`（待开发） | 封面图、海报、配图生成 | .png/.jpg 文件 |
| `scheduler`（待开发） | 定时任务管理（竞品监控、定时推送） | cron 任务 |

**MCP（Model Context Protocol）**——标准化的外部工具接入协议，让第三方能力即插即用：
- 每个 MCP Server = 一个独立的能力模块（如：聚光投放、小红书发布、企微群发）
- Agent 通过 MCP 协议发现并调用这些能力，无需硬编码
- 用户/社区可以贡献新的 MCP Server 来扩展产品能力

### 成品产出（而非 Markdown / JSON）

Skills + Bash 的关键意义：**团队成员交付的是成品，不是草稿**。

| 用户说 | 当前产出 | 有 Skills 后的产出 |
|--------|---------|-------------------|
| "帮我做个竞品分析报告" | Markdown 文本 | .xlsx 数据表 + .pptx 汇报 deck |
| "写一篇公众号文章" | Markdown 文本 | .docx 排版稿（可直接贴进编辑器） |
| "帮我做个活动方案" | Markdown 文本 | .pptx 方案 deck + .xlsx 预算表 |
| "做张小红书封面" | 文字描述 | .png 成品图（视觉模型生成） |
| "盯一下竞品最近发了什么" | 一次性搜索结果 | 定时监控 + 自动推送变更报告 |

### 记忆系统：无限上下文 + 团队共享

核心机制：**对话记忆向量化** → 突破单次对话的上下文窗口限制。

```
┌─────────────────────────────────────────────────┐
│                 记忆系统架构                       │
│                                                   │
│  实时对话 ──→ 滚动摘要 ──→ 向量化存入记忆库       │
│                                                   │
│  记忆库内容：                                     │
│  ├─ 历史决策（"上次定的账号人设是..."）             │
│  ├─ 爆文记录（"这篇笔记 10w 赞，因为..."）        │
│  ├─ 素材积累（"之前收集的竞品数据..."）             │
│  ├─ 用户偏好（"老板不喜欢用感叹号..."）            │
│  └─ 项目进展（"618 活动方案已确认..."）            │
│                                                   │
│  新对话开始时：                                   │
│  用户输入 ──→ 向量检索相关记忆 ──→ 注入上下文     │
│                                                   │
│  效果：                                           │
│  - 团队成员"记得"所有历史（不会反复问同样的问题） │
│  - 跨对话的项目连续性（上次聊到哪，这次接着来）   │
│  - 多人共享同一记忆库（团队协作场景）             │
└─────────────────────────────────────────────────┘
```

这意味着：
- **员工能自主获取最新信息**：联网 + 无头浏览器 + 定时任务 → 团队成员自己追热点、盯竞品、查数据
- **员工记得所有历史**：记忆向量化 → 过往决策、历史爆文、素材积累永不丢失
- **员工产出成品**：Skills（Excel/Word/PPT/图片）→ 交付的是可直接使用的文件，不是需要二次加工的文本

## 三、角色覆盖（11 个角色）

### A. 核心角色——直接替代（7 个）

#### 1. 文案 / 写手
- **替谁**：公众号写手、小红书笔记写手、自由撰稿人
- **场景**：写公众号文章、小红书图文、知乎回答、口播稿、种草文
- **靠什么**：LLM 对话 + 风格库仿写（学会用户的调性后持续产出）
- **上限**：对接发布 API 后可直接发布到平台

#### 2. 电商文案
- **替谁**：电商运营文案岗、详情页外包
- **场景**：商品标题优化、详情页文案、促销活动文案、买家秀引导
- **靠什么**：LLM 对话 + 卖点提炼 + A/B 变体生成
- **上限**：对接电商平台 API 后可直接上架/更新商品描述

#### 3. 编导 / 脚本策划
- **替谁**：短视频编导、脚本写手
- **场景**：60 秒短视频脚本、分镜脚本、口播逐字稿、直播脚本
- **靠什么**：结构化输出（钩子—内容—转化）+ 风格库

#### 4. 选题策划
- **替谁**：内容策划岗、选题会
- **场景**：热点追踪、竞品内容分析、选题日历、内容方向规划
- **靠什么**：联网搜索 + 趋势分析
- **上限**：无头浏览器 + 定时任务 → 自动监控热点并推送选题建议

#### 5. SEO 专员
- **替谁**：SEO 优化外包、关键词专员
- **场景**：关键词研究与布局、标题/标签优化、搜索排名监控
- **靠什么**：联网搜索 + 数据分析
- **上限**：无头浏览器定时抓取排名数据 → 自动生成优化建议

#### 6. 品牌策划
- **替谁**：品牌顾问、IP 人设策划
- **场景**：账号人设定位、品牌故事、slogan、品牌视觉调性建议
- **靠什么**：LLM 对话 + 风格库 + 竞品分析

#### 7. 活动策划
- **替谁**：活动运营、营销策划
- **场景**：线上活动方案、促销策划（双 11/618）、执行 SOP、裂变方案
- **靠什么**：方案生成 + 结构化 SOP 输出

---

### B. 辅助角色——纳入定位（4 个）

#### 8. 数据分析师
- **替谁**：数据运营、数据分析外包
- **场景**：内容效果分析、竞品数据对比、用户画像分析、投放 ROI 复盘
- **当前能力**：用户贴数据 → AI 解读、生成报告、给优化建议
- **上限**：bash + 无头浏览器 → 直接登录平台后台抓数据 → 自动生成周报
- **协作点**：⚠️ 登录平台后台需用户授权确认

#### 9. 投流师
- **替谁**：信息流优化师、广告投放外包
- **场景**：投放素材文案、定向建议、A/B 文案变体、ROI 分析
- **当前能力**：文案生成 + 策略建议
- **上限**：对接聚光/巨量引擎 API → 自动创建广告计划、调整出价、暂停低效计划
- **协作点**：⚠️ 创建/修改广告计划需用户确认；预算变更需用户确认

#### 10. 新媒体运营
- **替谁**：账号运营、内容分发专员
- **场景**：内容排期、平台规则解读、话术模板、跨平台分发策略
- **当前能力**：建议 + 模板 + 规则解读
- **上限**：对接平台 API / 无头浏览器 → 代发内容、自动排期、评论区自动回复
- **协作点**：⚠️ 发布内容需用户确认

#### 11. 社群运营
- **替谁**：社群运营专员、私域运营
- **场景**：社群内容策划、欢迎语/日常话术、活动文案、用户分层建议
- **当前能力**：内容策划 + 话术模板
- **上限**：对接企业微信 API → 自动群发、标签管理、SOP 消息推送
- **协作点**：⚠️ 群发操作需用户确认

## 四、人机协作模型

### 原则：Agent 自主执行 + 关键节点人工确认

不是所有操作都需要确认。确认机制分两级：

#### 无需确认（Agent 自主完成）
- 文案生成 / 改写 / 润色
- 联网搜索 / 信息收集
- 数据分析 / 报告生成
- 风格库检索 / 仿写
- 文件读取 / 项目浏览
- 竞品监控（只读）

#### 需要用户确认（有现实后果的操作）
- 🔴 **发布类**：发布内容到任何平台（小红书/抖音/公众号等）
- 🔴 **花钱类**：创建/修改广告计划、调整出价、设置预算
- 🔴 **群发类**：社群群发消息、批量私信
- 🟡 **写入类**：覆盖/删除用户已有文件（新建文件可自主）
- 🟡 **授权类**：首次登录第三方平台、授权 API 访问

### 确认 UI

不是当前的"结构化问卷菜单"，而是类似团队成员的自然语言确认：

```
❌ 当前（IDE 风格）：
[需要你确认]
你想测试什么？
- A 测试 IDE 是否能看到当前文件
- B 测试某个功能或工作流
- C 其他

✅ 期望（团队成员风格）：
"广告计划草稿已经拟好了：
  - 计划名：618 种草笔记推广
  - 日预算：500 元
  - 定向：18-35 女性，美妆兴趣
  确认投放吗？我先不动，你说投我再提交。"
```

## 五、对系统 Prompt 的影响

### 已完成的改动

1. **Agent 身份重写** ✅：从"写作 IDE Agent"→ "用户的 AI 内容团队（一个人的内容团队）"，声明 11 个角色
2. **Router 智能化** ✅：Intent Router 不再默认 ask_clarify；短消息/模糊消息走 respond_text
3. **确认机制重做** ✅：从"所有主动行为都确认"→"只在有现实后果时确认"（发布/花钱/群发/删除），且用自然语言
4. **路由表简化** ✅：移除 `visibility_contract`（IDE 概念）和 `debug` 路由

### 后续演进方向：主 Agent + Subagent 架构

当前 11 个角色挤在一个 prompt 里，存在"角色稀释"风险。后续采用 **Subagent 模式**拆分：

```
用户消息
  ↓
主 Agent（负责人/调度员）
  ├─ 自己处理：简单对话、短答、任务理解与分解
  ├─ 派发给 Subagent（各有专属 prompt + 工具集 + 模型）：
  │   ├─ 文案写手 Agent    — 风格库 + 仿写闭环 + lint
  │   ├─ 选题策划 Agent    — 联网搜索 + 热点分析
  │   ├─ 数据分析 Agent    — 数据解读 + 图表 + 报表生成
  │   ├─ SEO Agent         — 关键词研究 + 标签优化
  │   ├─ 投流顾问 Agent    — 素材文案 + 策略 + API 对接
  │   ├─ 电商文案 Agent    — 详情页 + 标题 + 促销
  │   ├─ 编导/脚本 Agent   — 视频脚本 + 分镜
  │   ├─ 品牌策划 Agent    — 人设 + 品牌故事 + slogan
  │   ├─ 活动策划 Agent    — 方案 + SOP + 预算
  │   ├─ 运营 Agent        — 排期 + 话术 + 社群
  │   └─ ...按需扩展
  └─ 汇总结果返回用户
```

**技术路径**：基于 Anthropic Claude Agent SDK（[Python](https://github.com/anthropics/claude-agent-sdk-python) / [TypeScript](https://github.com/anthropics/claude-agent-sdk-typescript)）的 `AgentDefinition` 实现：

```typescript
// 示例：每个角色一个 AgentDefinition
{
  "copywriter": AgentDefinition({
    description: "文案写手：写公众号/小红书/口播稿，擅长风格仿写",
    prompt: "你是专业文案写手...",
    tools: ["kb.search", "lint.copy", "lint.style", "doc.write"],
    model: "sonnet"
  }),
  "topic_planner": AgentDefinition({
    description: "选题策划：追热点、分析趋势、制定内容日历",
    prompt: "你是选题策划专员...",
    tools: ["web.search", "web.fetch", "time.now"],
    model: "haiku"  // 选题检索用便宜模型就够
  })
}
```

**为什么选 Subagent 而非 Agent Teams**：

| | Subagent | Agent Teams |
|---|---------|-------------|
| 成本 | 低（按需调起） | 高（每个队友是独立实例） |
| 复杂度 | 主 Agent 调度即可 | 需要邮箱/任务列表/文件锁 |
| 我们的场景 | 用户→负责人→派活→交付 | 更适合多人协作写代码 |

**实施节奏**：先把主 Agent 单 prompt 跑通 → 识别哪些角色的专业度不够 → 逐个拆为 Subagent

**详细规范**：→ [Sub-agent 架构规范 v0.1](sub-agent-architecture-v0.1.md)（标准接口、中转/广播两种通信模式、群聊 UI、设置机制、工程路径）

## 六、竞品参考

| 产品 | 定位 | 我们的差异 |
|------|------|-----------|
| ChatGPT / Claude | 通用 AI 助手 | 我们专注内容赛道 + 风格库（"会你风格"） |
| Jasper | AI 营销文案工具 | 我们覆盖更多角色（不只文案） + 中国平台生态 |
| 秘塔写作猫 | AI 写作工具 | 我们有执行能力（bash/API）不只是生成文字 |
| Notion AI | 文档内 AI 助手 | 我们不依附编辑器，对话即工作台 |
| Coze / 扣子 | AI Bot 构建平台 | 我们开箱即用不需要搭建，面向内容场景预置角色 |

核心差异化：
- **风格库**（学会你的调性，不是通用 AI 腔）
- **记忆系统**（无限上下文，记得所有历史决策和素材）
- **成品产出**（交付 Excel/Word/PPT/图片，不是 Markdown）
- **执行能力**（bash/MCP/API，不只是建议，能直接干活）
- **内容赛道垂直**（11 个角色预置 + 中国平台生态适配）
