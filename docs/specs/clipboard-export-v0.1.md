## Clipboard Export v0.1（复制到小红书/公众号/知乎/飞书 + 不影响 Agent）

### 0) 目标

- **核心目标**：让写作者在 Desktop 中“选中一段 → 一键复制”，粘贴到目标平台时尽量保留可读排版（段落、标题、加粗、列表、引用、代码块等）。
- **平台优先级**：小红书 → 公众号 → 知乎 → 飞书。
- **不影响 Agent**：任何“复制/导出/样式预设”变化，都不能改变 Agent 的读写契约与行为。

---

### 1) 约束与不变性（Invariants）

- **文件真源不变**：项目文档仍以可见文件落盘（当前默认 `.md`）。
- **Agent 契约不变**：
  - Agent 通过 `doc.read/doc.write/doc.applyEdits` 等工具读写的仍是**纯文本内容**（Markdown 文本）。
  - 剪贴板导出只影响“复制到外部平台”的交付链路，不改变文件内容结构、也不引入新的真源格式。
- **剪贴板必须写入兜底文本**：即便富文本写入失败，也要保证 `text/plain` 可粘贴。

---

### 2) 用户交互（v0.1）

- **复制富文本**
  - 复制范围：**优先选区**，无选区则复制全文。
  - 输出：写入 `text/html` + 同步写入 `text/plain`（兜底）。
  - 平台预设：小红书/公众号/知乎/飞书（默认小红书）。

---

### 3) 富文本 HTML 输出策略（v0.1）

#### 3.1 为什么要“内联样式”

多数内容平台会清洗 `<style>`、部分会清洗 `class`。因此 v0.1 采用：
- **基础标签**（`p/h1/h2/h3/strong/em/ul/ol/li/blockquote/pre/code/hr/a/br`）
- **内联样式**（`style="..."`）保证段落间距、字号、行距在粘贴时尽量保留。

#### 3.2 平台预设差异（v0.1 只做小幅差异）

- **小红书**：段落间距更大、行距更松、标题更醒目。
- **公众号**：风格更克制，减少花哨背景；强调可读性。
- **知乎**：更偏正文阅读排版，标题层级清晰。
- **飞书**：更偏办公文档排版，字号略小。

> 说明：v0.1 只做“排版参数”层级的微调（font-size/line-height/margins），避免引入平台特有 hack。

---

### 4) 验收清单

#### 4.1 基础
- 选中一段文本 → 点击“复制富文本” → 粘贴到任意纯文本工具：至少粘贴出 `text/plain`。
- 无选区 → 点击“复制富文本” → 复制全文生效。
- 复制不触发文件写入；不会改变当前文件内容；不会影响 Agent 的 `doc.read` 输出。

#### 4.2 平台粘贴（手工回归）
- **小红书**：段落分隔清晰；标题加粗；列表与引用可读。
- **公众号**：段落不挤；标题层级清晰；引用块不乱。
- **知乎**：段落与标题层级清楚；列表正常。
- **飞书**：段落与列表可读；代码块可读。

---

### 5) 回滚方案

- 删除/隐藏“平台预设 + 复制富文本”按钮，不影响编辑器核心能力与 Agent。
- 保留“复制富文本写入 `text/plain` 兜底”作为最小可用交付链路（纯文本可用于口播）。
