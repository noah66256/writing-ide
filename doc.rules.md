## Doc Rules（文档规则）

> 这是"项目级长期规则"，跨 Run 生效；用于约束写作目标、风格与禁用项，防止越写越跑偏。
> 修改规则必须走"提案→确认→写入"，并保留版本可回滚。

### 写作定位
- 本项目是**对话驱动的 AI 内容团队**，一切以内容产出与交付为中心。

### 默认风格与口吻（可按项目改）
- 语气：清晰、直接、结构化。
- 句长：中短句为主，避免拖沓。
- 禁用：空泛口号、无依据的数据/年份、强行营销腔（除非目标明确需要）。

### 平台画像优先级
- 默认先明确平台画像（feed 试看型 / 点选搜索型 / 长内容订阅型），再写作与改写。

### 引用与事实
- 涉及事实/数据/年份：尽量给来源；不确定就提示风险，不要编造。

### 输出格式约束
- 优先输出结构（outline）再展开正文。
